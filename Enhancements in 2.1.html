4th Dimension Compatibility

 

ResourceEditor Version 2.1 works with 4D 6.7.  It also works well with 4D 6.8.x and 4D 2003 (open the .4DB file with your copy of 4th Dimension to convert the structure to a newer version).

 

ResourceEditor has not been tested with earlier versions of 4th Dimension.

 

Compiling ResourceEditor

 

To recompile ResourceEditor, you may set your compiler options to “All Variables are Typed”.

 

Main Features of Orchard Software’s ResourceEditor

 

·    Translating from a parent to a child file

·    Editing a resource file

·    Creating a new resource file

·    Pig Latinizing a parent file

·    Exporting resource lists

·    Importing resource lists

·    Asian mode

·    Printing resource lists

·    Basic Find and Replace functions

·    PICT resource processing (New)

 

Improvements in Version 2.0

 

·    Added window icons and changed the title of the 4D windows, using Orchard Software’s free Win32API plugin.

·    Added logic to limit the size of the file dropdowns of previously viewed files.

·    Combined the Open and New dialogs into a single Open window.

·    The Open window comes up when Resource Editor opens instead of a blank 4D window.

·    Edits made in the Edit window no longer permanently change the resource file unless the user clicks the Save button.

·    Added a Next Blank option to the Edit window.

·    Added a Changed Lists option to the Edit window.

·    Added the ability to remove the Duplicate Value message that can appear when editing a resource file.

·    Removed the New Resource Type option.

·    Added PICT resource processing.

·    Added edit window dialog view customization between PICT resource, parent and child file, and regular resource file editing.

·    Changed terminology:  Master File was changed to Parent File. 

·    The Administrator Mode setting has been removed.  It was set via the presence of an INI file, and was required in translate parent/child files.

·    The parent file no longer must be set to open a resource file.  A parent file should only be opened if translating from one file to another or if Pig Latinizing a file.

·    Rewrote the FON# option to eliminate bugs and increase usability.

·    Rewrote the printing function to allow continuous printing or page breaks between resource lists, and When printing a resource file, a footer is printed with the date, time, and page number.

·    Added Wait windows to indicate extended processing

·    Added a Types menu to quickly switch between resource types.  Removed the resource type window.  The STR# resource is opened by default for all resource files.

·    Rewrote the Find and Replace functions to fix bugs.

·    Rewrote the synchronization method between parent and child resource files.  Resource Editor asks the user if they would like to fill the copy of the child file with blanks or copies of parent file entries.  Synchronize does not permanently change either file.   The child file is only changed if the user clicks save in the edit window.

·    A new file can be blank or a copy of a parent file.

·    The Edit window is now resizable.

·    Fixed Drag and Drop in the Edit window.  Dragging down was off by one.

·    Made changes to several dialogs to make user entry more key-driven instead of mouse-driven.

·    The File and Resource menu options have changed.

·    Synchronizing is only done once when a parent and child file are opened.  It cannot be done later.

·    The Japanese option has been renamed to Asian Mode.

·    When Pig Latinizing a file, the values are copied from the parent, Pig Latinized, and copied into the child file, rather than Pig Latinizing the child file in place.

 

Improvements in Version 2.0.1 (10/23/2002)

 

·    Fixed a problem in 2.0 that made it impossible to create a new PICT resource.
